🎨 Full MTL Texture Support — GSoC 2026 Proof of Concept

By aakritithecoder · Processing Foundation · p5.js WEBGL

map_Kd textures multi-material graceful fallback path resolution

What this prototype demonstrates:
p5.js currently ignores map_Kd (and all other texture maps) in .mtl files, rendering models as flat solid colours. This prototype patches the MTL parsing pipeline to:

  • Parse map_Kd, map_Ka, map_Ks, map_Ns, map_d, map_Bump, norm directives
  • Resolve relative texture paths correctly (including Windows-style backslashes)
  • Async-load textures inside p5's preload() with deduplication
  • Gracefully fall back to Kd solid colour if a texture file is missing
  • Apply the correct texture per material during rendering

The full GSoC implementation extends this to GPU-efficient multi-material rendering using index buffer face ranges, setUniform() shader binding, and normal/bump map support via TBN matrix computation.

Key file: mtl-loader.js — shows the proposed parseMTL(), loadMTLWithTextures(), and drawModelWithMaterials() functions.
